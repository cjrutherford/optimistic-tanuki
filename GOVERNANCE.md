# Governance

## Stewardship

Optimistic Tanuki is maintained as a product portfolio repository. Direction is set by the maintainers responsible for product scope, platform integrity, and release posture.

## Decision Making

Maintainers review changes against four questions:

1. does the change strengthen or clarify the product surface?
2. does it fit the existing platform and workspace architecture?
3. does it keep operational and security posture acceptable for the affected surface?
4. does it improve the repository without creating avoidable maintenance drag?

## Change Acceptance

Changes are most likely to be accepted when they:

- solve a clear product or platform problem
- follow existing repository patterns
- include the right level of validation for the touched surface
- keep docs in sync with behavior and positioning

## Security And Public Claims

Public-facing claims should match implemented behavior. Security hardening, contributor guidance, and developer-facing documentation should land before those claims are amplified in marketing or developer outreach.

## Releases And Surface Area

Not every app, service, or package in the monorepo has the same release posture. Maintainers may stage product packaging, self-hosting guidance, npm publication, and operational support independently for each surface.

## Conduct

Contributors are expected to engage directly, respectfully, and in good faith. If a discussion blocks progress, maintainers make the final call so the repository can continue moving.
